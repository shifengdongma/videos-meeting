  视频会议管理系统 UI 重设计方案

  一、全局设计系统

  1. 设计目标

  在不改变任何原始业务功能、文本、数据、状态、投票信息、链接的前提下，将现有系统统一升级为：    

  - 专业企业级
  - 沉浸式
  - 清晰高层级
  - Glassmorphism 2.0
  - 深 Indigo 主色体系

  这是一次表现层重构，不是产品逻辑改造。

  ---
  2. 全局视觉语言

  色彩系统

  严格使用文档指定色值：

  - 主色 / Primary Indigo：#2E3A59
    - 用于主按钮、侧边栏、重点高亮、主操作
  - 待开始 / Success-Ready Teal：#1E9E6F
  - 进行中 / Active Amber：#FBC02D
  - 已结束 / Soft Crimson：#E57373
  - 页面浅背景 / Surface：#F4F6F8
  - 浅底主文本 / Text Primary：#1A1F3B
  - 深底文本 / Inverse Text：#FFFFFF

  材质语言

  - 所有核心卡片使用克制的玻璃拟态
    - 半透明白层
    - 轻度背景模糊
    - 柔和投影 0 4px 12px rgba(0,0,0,0.05)
  - 输入框、选中态、表单内嵌区域使用轻度新拟物内凹
  - 后台页面避免过度透明，保证可读性优先

  字体系统

  统一使用 Inter

  - H1：2.5rem / Bold
  - H2：2rem
  - H3：1.5rem
  - Body：1rem
  - Caption：0.875rem

  图标系统

  统一使用 Heroicons

  - 导航图标统一线性风格
  - 状态图标与数据卡图标保持一致视觉重量
  - 禁止使用 emoji 代替功能图标

  ---
  3. 全局布局规则

  应用壳层

  适用于页面 3 / 4 / 5 / 6：

  - 左侧固定 Indigo 深色侧边栏
  - 右侧为浅灰主内容区
  - 顶部保持标题区与用户操作区
  - 内容卡片之间保持统一间距与对齐系统

  卡片规则

  - 统计卡：强调数字与状态图标
  - 数据表格卡：更浅边界、更强悬浮层级
  - 右侧辅助面板：玻璃感更强，但信息层级更严格

  ---
  4. 全局交互规则

  严格覆盖文档要求：

  - 侧边栏切换：平滑过渡
  - 卡片 hover：轻微抬升 + 阴影增强
  - 按钮 hover：亮度略提 + 阴影微增
  - 按钮 press：轻微压下反馈
  - 统计卡片：入场时轻微上浮渐显
  - 所有交互保持企业级克制，不炫技

  ---
  5. 严格保留规则

  以下内容一律不得丢失：

  - 所有原始文字
  - 所有数字
  - 所有状态
  - 所有表格列与表格动作
  - 所有投票信息
  - 所有投票链接
  - 所有按钮与功能入口

  二、逐页重设计方案

  ---
  页面 1：登录页 UI-design/1.jpg

  1. 页面目标

  将当前登录页升级为更有品牌感、信任感、沉浸感的企业视频会议入口页，在提升高级感的同时保留所有原
  有登录内容。

  2. 布局结构

  采用全屏左右分栏布局：

  - 左侧 55%
    - 品牌展示区
    - 保留原有主标题与说明文案
    - 增加现代视频会议场景插画/抽象可视化背景
  - 右侧 45%
    - 居中玻璃拟态登录卡片
    - 保留用户名、密码、按钮等全部原内容

  3. 视觉处理

  - 左侧背景以深 Indigo 渐变为主，叠加柔和光感与视频网络连线意象
  - 右侧卡片使用半透明玻璃质感，增强专业感
  - 页面整体由“普通后台登录”升级为“企业协作平台入口”

  4. 组件级重设计说明

  - 原来的三个圆角能力项：
    - 稳定会议
    - 实时音视频
    - 在线表决

  升级为 3 张小型功能卡：
    - 每张卡配 Heroicons 图标
    - 保留原文案
    - 增加一句简短功能解释
    - 使用悬浮微交互强化可点击/可感知性
  - 登录卡内部：
    - 标题与辅助文案保留
    - 用户名/密码输入框改为轻内凹风格
    - 主按钮“登录进入系统”改为 Indigo 主按钮
    - 保留原有功能位置与表意

  5. 动效与交互说明

  - 左侧功能卡 hover：微抬升 + 边缘高光
  - 输入框 focus：内凹边界高亮
  - 登录按钮：
    - hover：亮度略升
    - press：轻压下沉
  - 整张登录卡可做轻微淡入上浮入场

  ---
  页面 2：注册页 UI-design/2.jpg

  1. 页面目标

  与登录页形成统一品牌壳层，让注册流程保持一致体验，同时明确区分“账号创建”动作。

  2. 布局结构

  完全沿用登录页的全屏分栏结构：

  - 左侧继续保留品牌展示区
  - 右侧为玻璃拟态注册卡

  3. 视觉处理

  - 与登录页使用同一视觉系统，避免风格断裂
  - 可通过右侧标题和主按钮状态强调“注册”语义

  4. 组件级重设计说明

  - 标题改为：注册账号
  - 主按钮改为：注册并进入系统
  - 其余文本、字段、说明全部保留
  - 若左侧也有能力说明区，则继续沿用功能卡表达

  5. 动效与交互说明

  - 与登录页一致，保证登录/注册切换时体验统一
  - 注册主按钮保留与登录按钮相同的交互反馈规则

  ---
  页面 3：会议中心 UI-design/3.png

  1. 页面目标

  将会议中心升级为一个更清晰、可扫描、可操作的核心运营仪表盘，让用户快速查看会议状态并进入会议。

  2. 布局结构

  采用标准后台壳层：

  - 左侧：Indigo 深色侧边栏
  - 右侧：
    - 顶部标题与描述
    - 统计卡区域
    - 会议表格区域
    - 右上角用户信息区

  3. 视觉处理

  - 侧边栏整体更深、更稳重
  - 当前选中项“会议中心”使用更明确高亮底色/高光条
  - 统计卡片统一玻璃拟态处理，但色彩按状态区分
  - 表格卡片降低视觉噪音，增强可读性和操作感

  4. 组件级重设计说明

  侧边栏

  - 保留 logo 与系统名称
  - 图标统一为 Heroicons
  - 当前项“会议中心”强化激活态
  - 其他导航项弱化但保持清晰识别

  统计卡片

  4 张卡片全部保留原数据，仅重做表现：

  - 全部：Indigo
  - 进行中：Amber
  - 待开始：Teal
  - 已结束：Soft Crimson

  设计重点：
  - 数字更大
  - 状态图标更明显
  - 标签与数值层级更清楚

  会议表格

  - 保留所有列、文本、状态、数据、操作
  - 表头更清晰
  - 行 hover 增加浅色高亮
  - “进入会议”从普通下拉/弱按钮风格升级为更明确的主操作按钮
  - 状态标签按全局状态色统一编码

  用户区

  - 右上角重组为：
    - 头像
    - 用户名
    - 退出按钮
  - 更像成熟 SaaS 后台头部操作区

  5. 动效与交互说明

  - 侧边栏切换：背景与高亮平滑过渡
  - 统计卡：淡入上浮
  - 表格行 hover：轻高亮
  - “进入会议”按钮 hover/press 明确反馈
  - 用户区下拉或退出按钮保留原有功能语义

  页面 4：直播中心 UI-design/4.png

  1. 页面目标

  让直播中心在无数据时也保持完整、专业、可信的产品感，避免空页面显得“未完成”。

  2. 布局结构

  延续页面 3 的后台壳层：

  - 左侧深色侧边栏
  - 右侧：
    - 顶部标题与描述
    - 3 张统计卡
    - 主内容空状态区域
    - 右上角“开启直播”按钮

  3. 视觉处理

  - 页面整体与会议中心统一
  - 通过空状态插画、文案排版与留白，提升“无直播数据”的完成度
  - 主操作按钮“开启直播”使用 Indigo 主按钮强化操作入口

  4. 组件级重设计说明

  - 当前侧边栏激活项为：直播中心
  - 顶部统计卡保留原数据，仅重做视觉
  - 空状态区：
    - 保留原有“无直播数据”相关文字
    - 加入更优雅、偏企业感的空状态插画
    - 插画可使用会议摄像头、直播流、信号波形等抽象组合
  - 右上角 开启直播 作为页面最高优先级按钮之一

  5. 动效与交互说明

  - 空状态插画保持轻微静态装饰感，不抢主信息
  - 开启直播 按钮 hover/press 明确
  - 统计卡保持统一淡入动效
  - 空态容器可有轻微玻璃光感

  页面 5：用户管理 UI-design/5.png

  1. 页面目标

  把当前用户管理页重塑为真正的“管理后台”页面，强调秩序感、权限感、数据管理感。

  2. 布局结构

  延续统一后台壳层：

  - 左侧深色导航
  - 右侧：
    - 页面标题区
    - 4 张统计卡
    - 用户表格/管理区

  3. 视觉处理

  - 页面标题明确使用：管理后台
  - 视觉风格偏理性、精确
  - 数据管理区域边界更清晰，表格层级更稳重

  4. 组件级重设计说明

  - 当前侧边栏激活项为：用户管理
  - 4 张统计卡保留原数据与文案，仅增强层级与色彩逻辑
  - 用户表格：
    - 保留所有列、用户信息、角色、状态、操作
    - 行 hover 提升可操作感
    - 表头更具权重
  - 角色调整：
    - 保留原功能
    - 表现上改成更现代的下拉控件样式
    - 选中态与展开态遵循统一设计系统

  5. 动效与交互说明

  - 下拉框 hover/focus 明确
  - 表格行 hover 更柔和
  - 状态标签统一色彩编码
  - 管理型操作按钮避免过度装饰，强调准确与清晰

  页面 6：会议控制台 UI-design/6.png

  1. 页面目标

  把会议控制台从“信息堆叠页面”升级为“沉浸式实时协作工作台”，让视频区域、共享区域、控制操作和投票
  信息层级更合理。

  2. 布局结构

  重新组织主视觉重心：

  - 左侧或主区域：本地视频 + 桌面共享
  - 中下层：浮动控制工具栏
  - 右侧：投票信息玻璃卡片

  布局原则：
  - 视频与共享区域占据主屏大部分面积
  - 控件从零散排布改为统一浮动控制条
  - 投票面板保留完整信息，但不再压缩主屏空间

  3. 视觉处理

  - 主工作区整体更沉浸
  - 视频区与共享区使用深色容器 + 轻玻璃边框
  - 浮动工具栏提升操作聚焦感
  - 右侧投票卡片采用高质感玻璃面板，但内容可读性优先

  4. 组件级重设计说明

  视频 / 共享区域

  - 本地视频窗口更大
  - 桌面共享区域更大
  - 两者应成为页面视觉主角
  - 保留原有区域含义与功能，不改业务逻辑

  浮动控制工具栏

  整合原有控制操作为统一工具栏，保留原功能：
  - 打开摄像头
  - 打开麦克风
  - 桌面共享
  - 发起投票

  工具栏特征：
  - 居中悬浮或贴近主工作区底部
  - 圆角胶囊式玻璃面板
  - 图标 + 文字组合更利于识别

  投票面板

  必须完整保留：
  - 进行中
  - 反对
  - 弃权
  - 赞成
  - 相关票数、状态、链接

  可优化点：
  - 将投票卡做成更清晰的分层信息模块
  - 标题、状态、票数、链接操作分级排布
  - 使用状态色提高可扫读性

  5. 动效与交互说明

  - 工具栏按钮 hover：高亮边框 / 背景发光
  - press：轻压反馈
  - 视频与共享区切换/聚焦时使用柔和过渡
  - 投票卡片内部链接与操作保留清晰可点击反馈
  - 整体动效必须轻、稳、专业

  1. 内容保留检查

  - 所有原始文本已保留
  - 所有原始数字已保留
  - 所有原始状态已保留
  - 所有原始表格列与操作已保留
  - 所有投票信息、票数、链接已保留
  - 所有原始功能入口未删减

  2. 全局风格检查

  - 主色为 #2E3A59
  - 状态色严格使用 Teal / Amber / Soft Crimson
  - 字体统一为 Inter
  - 图标统一为 Heroicons
  - 卡片使用克制的玻璃拟态
  - 输入框/选中态带轻度新拟物内凹效果

  3. 页面要求检查

  - 登录页采用全屏分栏
  - 登录页 3 个能力项升级为功能卡
  - 注册页标题为“注册账号”
  - 注册页按钮为“注册并进入系统”
  - 会议中心有 4 个状态统计卡
  - 直播中心有 3 个统计卡与优雅空状态
  - 用户管理标题为“管理后台”
  - 用户管理保留“角色调整”并升级视觉表现
  - 会议控制台的视频与共享区域已放大
  - 会议控制台控制按钮已整合为浮动工具栏
  - 投票面板信息完整无损

  4. 交互检查

  - 所有按钮有 hover / press 状态
  - 侧边栏有平滑切换反馈
  - 统计卡片具备轻量入场动效
  - 表格行 hover 清晰但不过度
  - 焦点态与可点击态足够明确
